STEP 02

WebGL 渲染管线 · 原创 Rain 着色器

首先搭建 WebGL 渲染器 Canvas 模块，实现以下管线：

  1. 初始化 WebGL 上下文，绑定全屏四边形（2 个三角形 = 4 顶点）
  2. 编译并链接顶点着色器 + 片段着色器
  3. RAF 渲染循环，每帧更新 uTime、uRes、uIntensity 三个 uniform

// 顶点着色器（极简）

attribute vec2 aPos;
void main(){
  gl_Position = vec4(aPos, 0.0, 1.0);
}

// 渲染循环核心

_loop() {
  const gl = this.gl;
  const p  = this.programs[this.current]; // 'rain' | 'snow'
  gl.useProgram(p.pg);
  
  const now = performance.now() / 1000 - this.startTime;
  gl.uniform2f(p.uRes, gl.canvas.width, gl.canvas.height);
  gl.uniform1f(p.uTime, now);
  gl.uniform1f(p.uIntensity, this.intensity);
  
  gl.drawArrays(gl.TRIANGLE_STRIP, 0, 4);
  this.animId = requestAnimationFrame(() => this._loop());
}

// Heartfelt 核心算法

Heartfelt 由 Martijn Steinrucken（BigWings）创作，包含三层雨水模拟：

StaticDrops — 静态附着水珠

随机分布在玻璃表面，通过 Saw 函数控制出现/消失的淡出节奏

DropLayer2 — 滑落雨滴

含完整物理：水滴主体 + 细尾迹 + 前缘小水珠，水平方向随高度正弦摆动

Drops — 多层合成

三层叠加（静态×2倍、Layer1、Layer2×1.85缩放），uIntensity 控制每层权重

法线 + 折射

用差分 vec2(cx-c.x, cy-c.x) 计算表面法线，扰动 UV 采样背景产生折射效果

背景替换

原版用 textureLod(iChannel0,...) 采样外部贴图；WebGL 无贴图，替换为 50 个程序化 bokeh 光斑函数

// 折射计算（expensive normals）
vec2 e  = vec2(.001, 0.);
float cx = Drops(uv+e,   t, sd, l1, l2).x;
float cy = Drops(uv+e.yx, t, sd, l1, l2).x;
vec2  n  = vec2(cx - c.x, cy - c.x);  // 表面法线偏移

float focus = mix(maxBlur - c.y, minBlur, S(.1, .2, c.x));
vec3  col   = background(UV + n, focus); // 折射后的背景坐标
STEP 04

黑屏 Debug · GLSL ES 未定义行为

Phoenix · 用户
刷新页面背景后页面还是黑的
GitHub Copilot
检查中……创建 test.html 测试页，在浏览器内显示 WebGL 报错信息。
Phoenix · 用户
绿色日志没有 compile error 或 link error。有显示，很暗的紫色光斑
🐛 根本原因
Heartfelt 原版大量使用 edge0 > edge1 的 smoothstep 调用（如 S(0.4, 0.0, d)、S(1.0, y, st.y)），在 GLSL ES 1.0（WebGL1/ANGLE/Windows）中这属于未定义行为，实际产生 NaN，一旦有一个像素计算出 NaN，通过加法传播导致整块区域变黑。
✅ 修复方案
用安全的手写 S() 函数替换宏定义，通过 abs(d) < 1e-7 防止除零，同时保持双向 smoothstep 都能正常工作。 
// ❌ 原版宏 — edge0 > edge1 时未定义行为
#define S(a, b, t) smoothstep(a, b, t)

// ✅ 修复版本 — 处理反向边和 0/0 除零
float S(float a, float b, float t) {
  float d    = b - a;
  float safe = abs(d) < 1e-7 ? 1e-7 : d; // 防止 0/0 = NaN
  float x    = clamp((t - a) / safe, 0.0, 1.0);
  return x * x * (3.0 - 2.0 * x);         // 标准三次多项式
}

适配 iResolution→uRes，iTime→uTime，移除 iMouse（静态视角），用 uIntensity 插值 LAYERS/DEPTH/WIDTH/SPEED 参数。WebGL1 循环需有常量上限，用 for(i<200) if(i>=LAYERS) break 技巧绕过。

// Just Snow 算法结构

Andrew Baldwin（baldand）的作品，每个迭代层独立计算雪花的位置和大小，通过视差（depth）产生空间感，DoF 效果让远近层模糊程度不同。

for (int i = 0; i < 200; i++) {
  if (i >= LAYERS) break;         // WebGL1 循环必须有常量上限
  float fi = float(i);
  
  float fi = float(i);
  vec2 q = uv * (1.0 + fi * DEPTH);  // 视差缩放
  q += vec2(
    q.y * (WIDTH * mod(fi*7.238917,1.) - WIDTH*.5), // 左右漂移
    SPEED * uTime / (1. + fi*DEPTH*.03)               // 竖直落速
  );
  
  // 菱形雪花 SDF
  vec2 s = abs(mod(q,1.)-.5+.9*r.xy-.45);
  float d = .6*max(s.x-s.y,s.x+s.y)+max(s.x,s.y)-.01;
  float edge = .005+.05*min(.5*abs(fi-5.-dof),1.); // DoF 边缘模糊
  acc += vec3(smoothstep(edge,-edge,d)*(...));
}

uIntensity 滑杆从左到右对应从轻雪（50层）到暴风雪（200层），DEPTH/WIDTH/SPEED 同步插值变化。

// 三轮亮度迭代

经过三轮调整，最终确定的背景参数：

Round 1 — 基础提亮

底色从 vec3(0.012,0.015,0.04) 提升到 vec3(0.02,0.03,0.08)，bokeh 乘数 0.12→0.18

Round 2 — 大幅提亮 + 后处理减弱

底色提升到 vec3(0.06,0.05,0.12)，增加 glow 层，bokeh 增至 40 个，紫色调改为冷色调，暗角 0.8→0.6

Round 3 — 玻璃通透感

底色 vec3(0.15,0.12,0.22)，双层辉光（城市暖光 + 天空冷光），bokeh 增至 50 个，加入 blur 感知的雾气漫射层，暗角降至 0.3

// 最终版背景 — 模拟雾玻璃透光质感
vec3 col = mix(vec3(.15,.12,.22), vec3(.22,.20,.35), uv.y);

// 城市底部暖色辉光
float glow = smoothstep(.8, .0, uv.y);
col += glow * vec3(.25,.18,.12) * 0.6;

// blur 越大整体越亮 (模拟 mip 采样效果)
col += vec3(.04,.035,.06) * blur * 0.3;

// 极轻暗角
col *= 1.0 - dot(uvV, uvV) * 0.3;

// 三层音频架构

Layer 1 · 呼吸底音

136.1Hz（地球年共振 / OM频率）+ 142.1Hz，产生 6Hz Theta 差拍，对应脑波冥想状态。每个声部有 0.04Hz LFO 微漂。

Layer 2 · 颂钵敲击

222Hz 基频 + 2.76× + 5.2× 泛音，各自独立指数衰减（6s/4s/2.5s）。随机 7-16 秒触发，模拟真实敲击节奏。

Layer 3 · 混响空间

双延迟线 1.8s + 2.7s，各带低通滤波反馈（0.45/0.35 反馈量），模拟石室混响拖尾。

// 颂钵敲击 — 三个泛音独立衰减
const partials = [
  { f: 222.0,            vol: 0.18, decay: 6.0  },  // 基频
  { f: 222.0 * 2.76,    vol: 0.07, decay: 4.0  },  // 第一泛音
  { f: 222.0 * 5.20,    vol: 0.025, decay: 2.5 },  // 第二泛音
];

partials.forEach(({ f, vol, decay }) => {
  const env = ctx.createGain();
  env.gain.setValueAtTime(0, now);
  env.gain.linearRampToValueAtTime(vol * ambientLevel, now + 0.012); // 急攻
  env.gain.exponentialRampToValueAtTime(0.0001, now + decay);        // 缓衰
});

// 随机定时 7-16 秒触发
const tick = () => {
  this._strikeBowl();
  this._bowlTimer = setTimeout(tick, 7000 + Math.random() * 9000);
};
🔊 科学依据
Theta 差拍（6Hz）：双耳差拍（Binaural Beat）是神经科学中研究较多的技术，左右耳输入相差 4-8Hz 时，大脑会"感知"到一个同频率的差拍振荡，与 Theta 脑波（与冥想、REM 睡眠相关）同频。
136.1Hz：基于一年 365.25 天的地球公转周期计算出的音频频率，在东西方冥想传统中常被称为"OM 频率"。
